# 样例 - 示例功能计划

## 目标

让玩家更快理解当前目标，并让 HUD 的反馈与新手节奏对齐。

## truth 与上下文

- 必读 wiki：
  - [[00 Wiki/Concepts/系统 - 核心循环]]
  - [[00 Wiki/Rules/规则 - 新手引导节奏]]
  - [[00 Wiki/Surfaces/界面 - 主HUD]]

## 实施顺序

### Phase 1 - 收口当前信息入口

- 明确 HUD 需要持续显示哪些状态
- 成功时应该看到：HUD 入口职责与 wiki 对齐
- 验收方式：定向 UI 验证
- 可能要回写哪些页面：主HUD、核心循环
- 什么情况必须停：如果 HUD 需要承担完整教学

### Phase 2 - 对齐新手节奏

- 让早期目标提示与引导规则一致
- 验收方式：新手流程验证
- 可能要回写哪些页面：新手引导节奏
- 什么情况必须停：如果涉及核心循环改写

## Acceptance Phase

- 对照：wiki truth
- 机器验收：最小可行测试与界面检查
- 需要人类试玩关注的点：前十分钟是否更清楚
