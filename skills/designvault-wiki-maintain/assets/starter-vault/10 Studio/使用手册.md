# 使用手册

这页只讲日常怎么用，不讲设计原则。

## 先看哪里

- 想看当前设计真相：进 [[../00 Wiki/Home|00 Wiki]]
- 想看长稿、实施计划、验收：进 [[Home]]
- 想快速扫当前工作状态：进 [[Dashboards/看板 - Studio 总览]]
- 想新建页面：看 [[Obsidian Tooling]]

## 最短工作流

1. 用 `/design` 先通过提问把设计收敛到 `Execution Plan`
2. 用 `/execute` 自动跑完整个 plan
3. 自己去游戏或应用里试玩
4. 发现问题时再开 `/bug`

## 三种通道怎么选

- `/design`
  - 适合新功能、重构、设计收敛；默认先 grill 提问，再产出 `Longform + Wiki + Execution Plan`
- `/execute`
  - 适合已有稳定 plan 的自动实施；从 plan 跑到验收、回写和执行日志
- `/bug`
  - 适合实现 bug、小范围修复、观察中发现的问题；必要时升级回 `/design`

## 每类页面什么时候建

- `设计长稿`
  - 单线程持续推敲设计时使用
- `Wiki 页面`
  - 当前稳定设计真相
- `Execution Plan`
  - 完整 implementation plan，用于 `/execute`
- `Execution Log`
  - 一次 `/execute` 对应一页，用于 phase handoff、验收和回写记录
- `Change Review`
  - 记录实现结果、偏差、验收和后续回写

## Dashboard 默认看什么

- 最近完成
- 待我试玩
- 待决策
- 最近回写

## 日常命令

如果你把维护脚本装到了 skills 目录，常用命令是：

```powershell
pwsh -File <path-to-designvault-wiki-maintain>\scripts\wiki_search.ps1 -VaultRoot <vault-root> -Query "Shopping"
pwsh -File <path-to-designvault-wiki-maintain>\scripts\wiki_status.ps1 -VaultRoot <vault-root>
pwsh -File <path-to-designvault-wiki-maintain>\scripts\wiki_lint.ps1 -VaultRoot <vault-root>
pwsh -File <path-to-designvault-wiki-maintain>\scripts\wiki_suggest_writeback.ps1 -VaultRoot <vault-root> -SourcePath ".\10 Studio\Execution Plans\计划 - My Task.md"
pwsh -File <path-to-designvault-wiki-maintain>\scripts\change_review_draft.ps1 -VaultRoot <vault-root> -Title "Change Review - My Task" -SourcePath ".\10 Studio\Execution Plans\计划 - My Task.md"
```
