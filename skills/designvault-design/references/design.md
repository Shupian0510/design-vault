# /design

Use `/design` to move from unstable discussion to one executable plan.

## Default Sequence

1. Read the smallest relevant wiki pages first.
2. Enter a `grill me` loop instead of writing durable artifacts too early.
3. Summarize what is stable, what is still open, and which assumptions are risky.
4. Write or update a longform draft only when durable synthesis is useful.
5. Promote stable content into wiki truth pages.
6. End with one implementation plan.

## Question Quality Bar

- Ask questions that materially change gameplay, UX, rules, formulas, AI behavior, scope, or implementation order.
- Prefer small batches of high-signal questions.
- If the user has already answered enough, stop asking performative questions and synthesize.

## Output Contract

The final result should make these things clear:

- stable truth
- remaining open questions
- risky assumptions
- affected wiki pages
- implementation concerns
- the final execution plan
